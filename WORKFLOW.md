# 砚·选股与分析工作流（v4）

> **职责边界**：本文件定义分析流程、交付流程、复盘制度与报告规范。
> - 数据接口细节：`DATA_SOURCES.md`
> - 投资判断框架：`PHILOSOPHY.md`
> - 记忆系统：`MEMORY_SYSTEM.md`

## 0. 开始前健康检查
1. `openclaw status`
2. `python3 scripts/memory/memory_manager.py list_stocks`
3. `python3 scripts/data/market_cache.py status`
4. 检查关键数据接口是否在线

若任一关键数据源失败：明确标注“数据暂不可得”，并下调结论置信度。

## 0.1 默认网络深搜规则（新增）
从现在起，**所有报告和分析类回答** 默认加入“网络深度搜索 + 本地数据交叉验证”流程，除非用户明确要求只用本地数据。

最低执行要求：
1. **本地数据先行**：优先跑工作区脚本获取行情、财务、资金、技术面、公告等结构化数据。
2. **网络深度补证**：使用 `web_search` / `web_fetch` 检索并补充以下至少一类外部信息：
   - 最新公告 / 公司动态
   - 行业催化 / 政策 / 海外映射
   - 券商、媒体、研究机构公开信息
   - ETF / 指数成分、基金资料、官网说明
3. **交叉验证**：明确区分
   - 本地脚本事实
   - 外部网页事实
   - 媒体观点 / 市场传闻
4. **冲突处理**：若本地数据与外部搜索冲突，必须显式写出：
   - 哪些字段冲突
   - 当前优先信任哪一源
   - 这会如何影响结论置信度
5. **输出落地**：在正式报告或深度回答中，默认体现“数据 → 搜索信息 → 交叉验证 → 结论”的链条，禁止只堆本地数据或只罗列搜索结果。

## 2. 证据链验证（Validate）
对候选标的，至少完成四层验证：
1. **技术/产品可行性**：业务是什么，靠什么赚钱
2. **商业落地**：订单、客户、产能、交付与催化
3. **财务质量**：盈利、ROE、现金流、负债、成长性
4. **估值位置**：PE/PB/PS/PEG 与历史分位、同业对比

任一关键层缺证据：降级为“观察”，不进核心池。

### 2.1 深度思考与真实性审查（新增，默认强制）
从现在起，所有分析不得停留在“数据罗列 + 结论摘要”。必须至少补做以下 6 类深度判断中的 3 类；若是深度报告或用户点名“深度分析”，则原则上 6 类都要覆盖：

1. **宣传口径审查**
   - 公司/基金/媒体怎么说？
   - 这些说法是事实、推断，还是营销表述？
   - 产品语言和实际进展是否匹配，是否存在明显夸大？

2. **进展真实性审查**
   - 所谓“合作、订单、落地、临床、量产、AI赋能”等表述，具体落到了哪一步？
   - 是签约、试点、样机、论文、公告，还是已经形成稳定收入/交付？
   - 若只看到概念和愿景，必须明确写“尚未证明商业化兑现”。

3. **财务正常性审查**
   - 财务数据是否互相匹配：收入、利润、现金流、负债、资本开支、回购、存货、应收等是否协调？
   - 若出现高利润低现金流、扩张过快、负债异常、估值口径失真等情况，必须展开解释，不得一笔带过。

4. **市场态度审查**
   - 市场为什么给当前价格/估值？
   - 当前是乐观不足、过度乐观，还是已提前交易？
   - 市场不买账的原因是什么？这些原因合理还是错杀？

5. **技术前景与产业位置审查**
   - 技术是不是伪需求、早期概念，还是已进入可验证阶段？
   - 公司在产业链里卡的是核心环节还是边缘环节？
   - 未来天花板来自技术扩散、政策扶持、客户绑定，还是只停留在想象空间？

6. **反方视角审查**
   - 如果当前判断是错的，最可能错在哪里？
   - 哪些证据是在支持判断，哪些证据其实在反驳判断？
   - 必须主动提出至少一个“最强反例”并回应。

7. **反驳后的继续举证（新增，默认强制）**
   - 不能只提出质疑或反驳，而不继续核实。
   - 一旦提出“可能夸大 / 可能不实 / 可能过度营销 / 可能未兑现”等判断，必须继续通过 `web_search` / `web_fetch` 寻找：
     - 反例
     - 第三方佐证
     - 同行业对比案例
     - 官方/论文/媒体的独立来源
   - 例如：若公司声称“全球首台/全球领先”，必须尽量搜索行业内其他公司、历史产品、论文或新闻，判断该说法是否站得住；若无法证伪或证实，必须明确写“目前仅见公司自述，外部未找到足够独立证据”。

8. **人员背景审查（新增，默认强制）**
   - 对公司深度分析时，默认研究核心人员背景：创始人、董事长、CEO、首席科学家、核心技术负责人。
   - 至少回答：
     - 这些人来自哪里（高校 / 医院 / 大厂 / 产业背景）
     - 是科研型、产业型，还是资本/销售型团队
     - 履历是否支持公司当前叙事
     - 团队背景与公司产品方向是否匹配
   - 若团队背景信息很少，也必须明确标注“公开可验证信息有限”。

9. **公开尽调（公司企业信息任务默认强制）**
   - 凡涉及公司、非上市公司、创业公司、企业主体、股权结构、工商信息、资质、合作、融资、司法风险等任务，默认加入公开尽调步骤。
   - 默认搜索与核验范围包括：
     - 国家企业信用信息公示系统
     - 公司官网
     - 新闻稿 / 融资稿
     - 招投标 / 政府项目文件
     - 法院 / 知识产权公开信息
     - 媒体、投资机构、数据库页面
   - 目标不是简单罗列链接，而是形成结构化尽调判断：
     1. 主体是否真实存在、名称是否一致
     2. 公开股东 / 关联主体 / 投资方线索是否能互相印证
     3. 所谓合作、融资、项目、资质是否有独立来源支持
     4. 是否存在司法、失信、知识产权、监管或信息披露风险线索
     5. 哪些信息已被证实，哪些仍停留在公司自述或媒体转述

输出要求：
- 不能只写“公司不错/行业很好/未来可期”这类空判断。
- 必须把“为什么可能是真的”与“为什么可能是夸大的”同时摆出来。
- 提出反驳后，必须继续搜索举证，而不是停在怀疑层面。
- 公司深度分析默认加入“核心人员背景”一节。
- 公司企业信息类任务默认加入“公开尽调”一节，至少呈现：主体核验、股权/关联线索、合作/融资核验、风险线索、信息置信度分层。
- 结论必须体现思考深度，而不是把公开材料换种说法复述。

## 3. 交易确认（Trade Plan）
1. 技术面：均线、MACD、RSI、量价结构
2. 资金面：主力资金、北向、筹码、大单行为
3. 情绪面：市场广度、热度、板块扩散度
4. 形成执行参数：
   - 建仓区间
   - 仓位比例
   - 止损位
   - 失效条件

## 4. 交付（Deliver）
- 长文分析、复盘、周报：先生成 Markdown，再转 PDF
- **复杂分析 / 深度分析 / 大篇幅分析：默认以 PDF 作为正式交付格式**；聊天内可附简短结论，但正式成品优先发 PDF
- **定时任务中的“撰写/成稿”阶段默认交给 Claude 完成**；本 agent 负责数据采集、证据链校验、风控约束、质检、归档与最终发布把关。若 Claude 不可用，必须明确降级说明，再由本 agent 直接成稿。
- 聊天内默认只给 3-6 条核心结论 + 风险提示
- 重大分析必须显式交代：数据覆盖、主导矛盾、关键变量、主裁决、推翻条件
- **日常报告（盘前 / 收盘复盘 / 盘中）**：不追求周报级篇幅，但必须把“数据 → 新闻/事件 → 预期 → 资金/价格”的链条写清，重点解释背后的关联、矛盾、统一性与失效条件，禁止只堆数据或只罗列新闻。
- **深度报告（周报 / 专题 / 板块深度）**：必须更深入利用已获取数据，要求多维交叉验证、历史对照、矛盾展开、多空对抗与统一裁决，禁止把已有数据只做表格堆砌。
- **周报硬规范**：见下方第 6 节完整规范。

## 6. 周报硬规范（砚·周度市场洞察）

> 适用对象：每周正式周报、补写周报、要求达到"定时周报级别"的深度报告。
> 目标：确保周报不是"长一点的群消息"，而是能经得起复盘、推敲与执行检验的正式研究成品。

### 6.1 定位与最低质量线
- 周报是当前工作区**最高级别**的例行研究成品。
- 不是简报，不是执行清单，不是短评拼接。
- 必须体现：数据覆盖广度、论证深度、多维交叉验证、对未来一周的可执行价值

#### 最低成品要求
- Markdown 正式成稿后再转 PDF
- 原则上不少于 **12000 字符** 的 Markdown 正文量级
- PDF 原则上不少于 **12 页**，通常应在 **12–20 页** 区间
- 若因数据不足未达到该深度，必须明确说明"为什么达不到"

### 6.2 强制章节骨架（不得缺失）

#### 6.2.1 数据覆盖说明
必须明确：
- 本周用了哪些数据源，数据覆盖到什么时间
- 哪些数据可信度高，哪些只能参考
- 有哪些数据缺口或冲突，当前信任源是什么
- 周报跨资产部分默认优先使用 `data/weekly_inputs/<date>/15_cross_asset.txt`

#### 6.2.2 全球市场概览与跨资产联动
必须覆盖并形成完整论证：
- 美股（至少：标普500、纳指、道指）
- 港股 / 日股 / 欧股中的关键市场
- 大宗商品（至少：原油、黄金、铜）
- 外汇（至少：美元指数、USD/CNH）
- 债券 / 利率（至少：美债 2Y / 10Y）
- 加密资产（至少：BTC、ETH 或风险偏好映射）
- 如有显著波动，解释这些资产如何传导到 A 股风格与板块定价

#### 6.2.3 宏观环境与政策
必须写清：
- 本周国内外宏观增量是什么
- 政策、利率、PMI、通胀、进出口等数据意味着什么
- 哪些是顺风，哪些是逆风
- 这些宏观变量如何改变市场风格与仓位选择

#### 6.2.4 A股市场结构
必须覆盖：
- 主要指数周度表现
- 风格强弱（成长 / 价值，大盘 / 小盘）
- 市场广度、成交、情绪温度
- 资金主线与弱线
- 是否存在"指数稳但结构差"或"结构强但指数钝化"之类的错位

#### 6.2.5 行业 / 赛道深度
至少深写 **3 个方向**，每个方向都必须覆盖：
- 本周发生了什么，资金怎么走
- 新闻 / 产业催化是什么
- 金融市场为什么会这样定价，这种定价是否可持续
- 历史上类似情形通常怎么演绎
- 多头逻辑 / 空头逻辑 / 我的裁决

> 禁止只罗列"净流入前五"。必须把"新闻 → 预期 → 资金 → 价格"的链条写出来。

#### 6.2.6 新闻与市场关系（单独成节）
必须把重要新闻做成"事件—影响映射"：
- 新闻内容是什么
- 影响的是增长、通胀、利率、风险偏好，还是产业预期
- 对全球市场与 A 股分别怎么传导
- 是一次性扰动还是中期变量

#### 6.2.7 历史对照
必须加入至少一个历史维度：
- 与过去 4–12 周相比，本周市场发生了什么变化
- 与历史类似环境相比，现在更像哪一类阶段
- 为什么这次不能简单照搬过去结论

#### 6.2.8 自选股 / 观察池周度跟踪
若观察池**非空**：每只标的至少覆盖价格表现、技术结构、估值/基本面、资金面/消息面/情绪面、本周验证了什么 / 证伪了什么、交易计划与失效条件。

**周报新增硬要求（K线详细复盘）**：每只观察池标的与至少 2 个关键指数，必须补充一段“详细 K 线技术面复盘”，至少覆盖：
- MA5 / MA10 / MA20 / MA60 的相对位置变化
- MACD / RSI / KDJ / 布林带中至少 3 项指标的最新状态
- 量价配合（放量突破 / 缩量反弹 / 放量下跌 / 缩量阴跌等）
- 关键支撑 / 压力位
- **变化原因解释**：不能只写“技术面转强/转弱”，必须回答为什么变（是放量、跌破均线、动能钝化、情绪退潮还是消息驱动）

若观察池**为空**：必须明确写"当前观察池为空"，不得擅自代入旧偏好或旧自选。可以补充候选方向，但必须标明只是候选。

#### 6.2.9 多空对抗与主裁决
必须写出：
- 多头最强论据 / 空头最强论据
- 双方最核心分歧点
- 我的最终裁决 / 推翻条件
- 如果我错了，最可能错在哪里

#### 6.2.10 世界系统性风险评估（新增固定板块）
必须覆盖并形成可执行结论：
- **风险指数汇总**：至少覆盖 VIX、MOVE（美债波动）、信用利差（IG/HY 或可得替代）、DXY、WTI、黄金、USD/CNH
- **机构运行状况**：银行/券商/资管/对冲基金的压力代理信号（股价异常波动、融资/流动性压力、赎回限制、监管处罚或重大突发）
- **风险事件追踪**：地缘冲突、流动性冲击、信用事件、政策突发；并标注一次性扰动 or 中期变量
- **风险温度分（0-100）**：给出绿/黄/橙/红分档、环比变化及触发原因
- **传导链与应对**：写清“海外风险源 → 汇率/利率/风险偏好 → A股风格/板块/仓位”的链条，并给出防守动作

#### 6.2.11 下周展望与行动框架
必须给出：
- 下周关键观察变量
- 哪些信号会加强判断 / 推翻判断
- 仓位、节奏、风控的原则性框架

### 6.3 多 agent 论战硬要求
周报必须使用多 agent 协作，且报告中必须详细阐述各方论证逻辑以及反驳逻辑。

**最低流程：**
1. **第一轮 — 分模块独立研究**（至少 5 个模块）：全球宏观与跨资产 / A股结构与资金 / 行业赛道与消息面 / 自选股或候选研究池 / 风险管理与反方视角
2. **第二轮 — 交叉质询**：至少一轮"互相挑错 / 反驳 / 补证据"，重点检查：是否观点当事实、是否单一新闻推导过强结论、是否忽略反例与历史对照、是否忽略风控
3. **第三轮 — 主编整合裁决**：写出分歧点，写出最终采用/放弃哪条链路，明确为什么这样裁决

**报告留痕**：最终周报必须单独有 `多agent讨论纪要` 或 `多空论战纪要` 一节。

### 6.4 表达要求
- 先结论，再证据，再推理，再风险，再行动
- 新闻不能只罗列，必须解释"为什么市场会这么交易"
- 数据不能只贴表，必须解释"数据在告诉你什么"
- 事实、观点、行动建议必须分开写
- 报告编排优先“可视化友好”：能用小表格、对照卡片、项目符号分层呈现的，不要挤成大段技术指标堆叠；技术面尤其要优先做成“指标/现状/含义”式直观展示
- 禁止在正式成品标题或小节标题中使用“详细版”“增强版”“修改版”“重构版”等过程痕迹命名

### 6.5 禁止事项
- 禁止为了凑篇幅堆砌无关新闻
- 禁止把旧关注方向当成当前观察池
- 禁止把低可信度数据写成确定性结论
- 禁止把周报写成 6 页左右的轻量版后仍声称达到定时周报级别

## 7. 收盘复盘制度

收盘复盘是每日最重要的环节——**不复盘的分析毫无价值**。

### 复盘 9 步骤

1. **预判回顾** — 读取当天盘前报告，逐条对照预判与实际，计算准确率，错了要搞清楚为什么错
2. **全天数据归档** — `save_reports` + `save_sentiment` + `save_signals`
3. **技术面复盘** — 自选股突破/跌破/形态、关键指数支撑压力验证、成交量异常。
   - **每日复盘新增硬要求**：必须对主要指数 + 观察池逐只做详细日K技术复盘，至少写出 MA5/10/20/60、MACD、RSI、KDJ、布林带、量价关系、关键支撑/压力，以及”今天为什么会这样变化”。
   - 禁止只写”技术面偏弱/偏强”这类一句话结论；必须解释变化驱动来自放量、缩量、均线得失、动能变化还是消息面催化。
4. **资金面复盘** — 当日资金流向对比、北向趋势连续性、龙虎榜异常机构
5. **情绪面复盘** — 盘前→盘中→收盘情绪变化、热股概念连续性、新题材持续性
6. **世界风险评估（固定板块）** — 汇总风险指数（VIX/MOVE/信用利差/DXY/WTI/黄金/USD-CNH）、机构运行状况（银行/券商/资管/对冲基金）与重大风险事件，给出当日风险温度分与对A股传导链
7. **研报变动** — 自选股评级变动、新增重磅研报、券商观点分歧/一致性
8. **经验沉淀** — 今天学到什么、什么方法有效/无效、下次改进
9. **认知更新** — 今天学到了什么新认知、哪些框架被验证或需调整

## 10. 不同金融产品分析规则（新增，默认强制）

### 10.1 先分产品，再分析
每次接到标的分析任务，先明确产品类型，并在输出中显式写出：
1. **产品类型**（个股 / ETF / 可转债 / 指数或跨境产品）
2. **适配框架**（对应本节子规则）
3. **核心风险来源**（估值、流动性、政策、风格、汇率等）

禁止把 ETF 当个股直接套模板，也禁止把个股逻辑直接嫁接到债券或商品产品。

### 10.2 各类产品最低分析骨架
- **个股（A股）**：商业模式与竞争力 → 财务质量 → 估值分位 → 资金/情绪/技术共振 → 交易计划
- **ETF（宽基/行业/主题/跨境）**：按 §10.3 ETF 专项框架逐项展开
- **可转债**：正股驱动 + 转股溢价率 + 纯债价值 + 条款风险（赎回/回售）+ 流动性
- **债券/货币类产品**：久期与利率敏感度 + 信用暴露 + 收益/回撤特征 + 流动性
- **商品/黄金/原油类 ETF**：现货-期货结构（升贴水）+ 美元与实际利率 + 库存/供需冲击 + 相关性漂移

若数据不足以覆盖该产品的最低骨架，必须明确写"数据暂不可得"并降低结论置信度。

### 10.3 ETF 专项深度分析框架（重点强化）
以后凡是 ETF 分析，默认按以下 **10 项清单**执行；深度分析场景原则上 10 项全覆盖：

1. **产品与发行信息**
   - 基金管理人、成立日期、跟踪标的、复制方式（完全复制/抽样/合成）
   - 最新规模（AUM）、份额变化、近阶段申赎方向
   - 费率结构（管理费/托管费/总费率）

2. **跟踪标的与指数方法学**
   - 指数编制机构、选样规则、权重规则、调仓频率
   - 前十大成分权重、行业集中度、主题纯度（是否“挂羊头卖狗肉”）
   - 是否存在风格漂移（名义主题与实际暴露不一致）

3. **成分质量与基本面透视**
   - 成分股盈利质量、估值分位、景气度分化
   - 龙头权重是否过高，是否存在单一权重股风险

4. **定价效率与交易成本**
   - 成交额、换手率、买卖价差（bid-ask spread）
   - 折溢价水平与稳定性
   - 跟踪误差与跟踪偏离（可得时）

5. **走势结构（绝对 + 相对）**
   - 1/5/20/60 日趋势、量价结构、关键支撑压力
   - 相对基准指数与同类 ETF 的强弱排序（相对强弱）

6. **资金面与拥挤度**
   - 份额增减、主力净流入、是否出现短期拥挤交易
   - 若可得，补充融资融券/期权情绪等杠杆信号

7. **消息面与情绪面传导**
   - 政策、产业催化、事件冲击如何传导到 ETF 定价
   - 舆情热度是否与资金流向一致，是否存在“情绪先行、基本面滞后”

8. **价值锚与比较基准（你提到的“锚定价值”）**
   - 对应成分的 PE/PB/股息率历史分位
   - 与同类 ETF 的费率、跟踪质量、流动性、估值暴露对比
   - 判断当前定价是合理、低估还是透支

9. **结构性风险清单**
   - 集中度风险、容量风险、流动性抽离风险、政策/监管风险
   - 跨境 ETF 额外加入汇率与时差风险

10. **可执行交易计划**
   - 建仓区间、仓位上限、止损/止盈、失效条件
   - 给出三情景（乐观/中性/悲观）触发条件，而非单一路径叙事

### 10.4 ETF 输出最低成品要求
ETF 正式分析（尤其是深度分析）必须同时给出：
- **事实层**：上面 10 项中可验证的数据与来源
- **观点层**：主导矛盾、当前交易的是预期还是现实
- **行动层**：仓位、节奏、风控、失效条件

并明确一句：**“若我判断错，最可能错在什么变量”**。

### 10.5 我额外补充的高价值检查项（新增）
除你提到的发行、标的、走势、消息、情绪、价值锚外，默认再补这 4 项：
1. **容量与冲击成本**：资金体量上来后，ETF 是否还能低冲击成交
2. **同类替代性**：是否有更低费率/更高流动性的替代 ETF
3. **风格暴露纯度**：名义上买“船舶”，实际是否被其他行业稀释
4. **再平衡事件风险**：指数调仓日前后，是否出现被动资金扭曲价格
